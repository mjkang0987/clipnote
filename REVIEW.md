# 코드리뷰 기준 (ClipNote)

> 자동 CI + 작성자(Claude)의 `/code-review`가 이 기준을 따른다. 리뷰 코멘트는 **한국어**로.

## 무엇을 "중요"로 볼지
아래만 **중요(반드시 지적)**. 나머지는 사소(nit) 이하:
1. **정확성/버그**: 로직 오류, null/undefined 처리, 경계값, 회귀.
2. **보안**:
   - 인증/인가: Supabase RLS·세션 검증이 우회되지 않는지, 사용자 데이터가 `user_id`로 스코프되는지.
   - 로그인/게스트(localStorage) 분기에서 타인 데이터 노출이 없는지.
   - 시크릿·토큰이 클라이언트 번들이나 로그에 노출되지 않는지(서버 전용 env 확인).
   - 입력 검증·공개 라우트 남용 방지.
3. **회귀**: 기존 흐름(로그인·공유 링크·미들웨어)을 의도치 않게 바꾸지 않는지.
4. **리팩토링·단순화**: 중복 로직, 재사용 가능한 부분, 불필요한 복잡도 — **제안(코멘트)** 으로만, 직접 커밋하지 않는다.

## 프로젝트 규칙
- **Conventional Commits 필수** (release-please가 버전 계산). PR 제목·커밋 prefix 확인.
- 웹 표준·접근성(WCAG)·시맨틱 HTML·네이티브 폼 우선.
- Next.js App Router: 서버/클라이언트 컴포넌트 경계, `use client` 남용 여부.

## 건너뛸 것 (지적하지 말 것)
- `*.lock`, `pnpm-lock.yaml`, 순수 문서(`*.md`).
- 개인 취향 수준의 네이밍/포맷팅(Tailwind 클래스 순서 등).

## 항상 확인
- `pnpm build`/타입체크가 깨질 타입 오류 없는지.
- Supabase 쿼리가 항상 사용자 스코프인지.
- 미들웨어(`middleware.ts`)·인증 흐름을 우회/변경하지 않는지.
