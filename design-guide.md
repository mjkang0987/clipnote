# ClipNote — design-guide.md

> 디자인 시스템의 source of truth. UI 작업 시 이 문서의 토큰·규칙을 따른다.
> 토큰은 `app/globals.css`의 CSS 변수 + Tailwind v4 `@theme`로 구현된다.

## 1. 디자인 원칙

- **콘텐츠 우선** — 화면의 주인공은 사용자가 클립한 콘텐츠. UI는 비켜선다.
- **그라디언트 아이덴티티** — 공유 카드의 랜덤 그라디언트가 브랜드의 얼굴. 그라디언트를 절제해서, 그러나 분명하게 쓴다.
- **빠르고 가볍게** — 한 화면에서 입력 → 결과. 단계와 클릭을 최소화.
- **접근성 기본값** — WCAG 2.1 AA. 시맨틱 마크업과 네이티브 폼 컨트롤 우선.

## 3. 그라디언트 프리셋 (공유 카드용)

공유 카드 배경은 아래 프리셋 중 **슬러그 기반 결정적(deterministic) 랜덤**으로 선택한다(같은 클립은 항상 같은 색).

| 이름 | 시작 | 끝 |
|------|------|-----|
| `sunset` | `#FF6B6B` | `#FFA94D` |
| `ocean` | `#4F8DFD` | `#6FE0C9` |
| `grape` | `#7C5CFC` | `#E879F9` |
| `forest` | `#0EA5E9` | `#22C55E` |
| `peach` | `#FB7185` | `#FDBA74` |
| `midnight` | `#4338CA` | `#7C3AED` |
| `mint` | `#06B6D4` | `#34D399` |

각도 기본 `135deg`. 카드 위 텍스트는 항상 흰색 + 약한 드롭섀도로 가독성 확보.

## 4. 타이포그래피

- **서체**: Pretendard (한글·라틴 공용). 폴백 `system-ui, sans-serif`.
- 숫자/코드: `ui-monospace` 계열.

| 토큰 | 크기 / 행간 | 용도 |
|------|------------|------|
| `display` | 40px / 1.15, 700 | 히어로 타이틀 |
| `h1` | 28px / 1.25, 700 | 페이지 제목 |
| `h2` | 22px / 1.3, 600 | 섹션 제목 |
| `body` | 16px / 1.6, 400 | 본문 |
| `small` | 14px / 1.5, 400 | 보조 |
| `caption` | 12px / 1.4, 500 | 라벨·메타 |

## 5. 간격 · 모서리 · 그림자

- **간격**: 4px 배수 스케일(Tailwind 기본). 컴포넌트 내부 12~16px, 섹션 간 32~64px.
- **모서리(radius)**: `sm 8px` · `md 12px` · `lg 16px` · `xl 24px` · `full 9999px`. 입력·버튼은 `md`, 카드는 `lg`.
- **그림자**: `shadow-soft` = `0 1px 2px rgba(0,0,0,.04), 0 8px 24px rgba(0,0,0,.06)`. 떠 있는 요소만 사용, 남발 금지.

## 6. 컴포넌트 규칙

### 버튼
- 1차(primary): `--brand` 배경 + 흰 글자, hover `--brand-strong`.
- 2차(secondary): 투명 배경 + `--border` 테두리.
- 최소 터치 영역 44×44px. 비활성은 투명도 + `cursor: not-allowed`.

### 입력 (URL · 제목 · 태그)
- 네이티브 `<input>`/`<textarea>` 사용. `--surface` 배경, `--border` 테두리, 포커스 시 `--brand` 링.
- 모든 입력에 연결된 `<label>` 필수(시각 라벨 또는 `aria-label`).

### 클립 카드
- `--surface` 배경, `lg` 모서리, `shadow-soft`. 썸네일 + 제목 + 설명 + 태그 + 날짜.

### 공유 카드 / OG 이미지
- 1200×630. 그라디언트 배경(3장) + 제목(크게) + 설명(2줄 말줄임) + ClipNote 워터마크.

### 태그
- `--brand-soft` 배경 + `--brand-strong` 글자, `full` 모서리, `caption` 크기.

## 7. 접근성 · 프런트엔드 표준 (필수)

- **태그 셀렉터 금지** — CSS는 id/class만. (Tailwind 유틸리티 클래스 사용)
- 시맨틱 HTML 우선: `<header> <main> <form> <button> <nav> <ul>` 등.
- 네이티브 폼 컨트롤(radio·checkbox·select·button) 우선, 불필요한 커스텀 UI 지양.
- 키보드 포커스 가시화: 모든 인터랙티브 요소에 `focus-visible` 링.
- 색만으로 정보 전달 금지(아이콘·텍스트 병행).
- 이미지 `alt`, 폼 오류는 텍스트로 명시.

## 8. 다크 모드

MVP는 라이트 기본. 토큰을 `prefers-color-scheme: dark`에서 재정의하는 구조로 두어 추후 손쉽게 확장(현재 다크 값 임시 정의, 정식 대응은 후순위).
